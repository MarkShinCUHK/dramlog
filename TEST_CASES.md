# MVP 6단계 기능 테스트 케이스

## 테스트 환경 준비
1. 개발 서버 실행: `npm run dev`
2. 브라우저 개발자 도구 열기 (F12)
3. Supabase 연결 확인 (`.env` 파일 설정 확인)

---

## 1. 반응형 디자인 개선 테스트

### 1.1 모바일 버튼 크기 최적화 (최소 44x44px)

**테스트 케이스 1: Header 모바일 메뉴 버튼**
- **조건**: 브라우저 창을 모바일 크기로 조정 (< 768px)
- **확인 사항**:
  1. 우측 상단 햄버거 메뉴 버튼 클릭
  2. 개발자 도구에서 Elements 탭 → 버튼 요소 선택
  3. Computed 스타일에서 `min-height: 44px`, `min-width: 44px` 확인
  4. 모바일 메뉴 내 링크들도 `min-height: 44px` 확인

**테스트 케이스 2: 폼 버튼 크기**
- **조건**: `/write` 페이지 접속, 모바일 크기로 조정
- **확인 사항**:
  1. 하단 "작성하기" 버튼 선택
  2. Computed 스타일에서 `min-height: 44px` 확인
  3. "취소" 버튼도 동일하게 확인

**테스트 케이스 3: 로그인/회원가입 버튼**
- **조건**: `/login` 또는 `/signup` 페이지 접속, 모바일 크기
- **확인 사항**: 제출 버튼이 `min-height: 44px` 이상인지 확인

---

### 1.2 태블릿 레이아웃 최적화

**테스트 케이스 4: 게시글 목록 그리드**
- **조건**: `/posts` 페이지 접속
- **확인 사항**:
  1. 브라우저 창을 640px로 조정 → 1컬럼 표시
  2. 768px로 조정 → 2컬럼 표시
  3. 1024px로 조정 → 3컬럼 표시
  4. 개발자 도구에서 `grid-cols-1 sm:grid-cols-2 md:grid-cols-2 lg:grid-cols-3` 클래스 확인

**테스트 케이스 5: 검색 결과 페이지**
- **조건**: `/search?q=테스트` 접속
- **확인 사항**: 동일한 그리드 레이아웃이 적용되는지 확인

---

### 1.3 데스크톱 최대 너비 및 여백 최적화

**테스트 케이스 6: 큰 화면 최대 너비**
- **조건**: 브라우저 창을 1280px 이상으로 확대
- **확인 사항**:
  1. `/posts` 페이지: `max-w-6xl xl:max-w-7xl` 적용 확인
  2. `/write` 페이지: `max-w-4xl xl:max-w-5xl` 적용 확인
  3. `/posts/[id]` 페이지: `max-w-4xl xl:max-w-5xl` 적용 확인
  4. `xl:px-8` 여백이 적용되는지 확인

---

### 1.4 모바일 입력 필드 최적화

**테스트 케이스 7: 입력 필드 터치 영역**
- **조건**: `/write` 페이지 접속, 모바일 크기
- **확인 사항**:
  1. 제목 입력 필드 선택 → Computed에서 `padding-top: 12px`, `padding-bottom: 12px` 확인 (py-3)
  2. 데스크톱 크기로 변경 → `padding-top: 10px`, `padding-bottom: 10px` 확인 (sm:py-2.5)

## 2. 에러 처리 개선 테스트

### 2.1 에러 페이지 개선

**테스트 케이스 8: 404 에러 페이지**
- **조건**: 존재하지 않는 페이지 접속 (예: `/posts/존재하지않는id`)
- **확인 사항**:
  1. 에러 코드 "404" 표시
  2. "페이지를 찾을 수 없습니다" 제목 표시
  3. 아이콘(원 안에 X) 표시
  4. "홈으로 돌아가기", "이전 페이지" 버튼 표시
  5. 버튼이 `min-h-[44px]` 적용 확인

**테스트 케이스 9: 403 에러 페이지**
- **조건**: 로그인 상태에서 익명 게시글 수정 페이지 접속
- **확인 사항**:
  1. 에러 코드 "403" 표시
  2. "접근 권한 없음" 제목 표시
  3. 경고 아이콘(삼각형) 표시
  4. 적절한 에러 메시지 표시

**테스트 케이스 10: 500 에러 페이지**
- **조건**: 서버 오류 발생 시 (임시로 서버 코드에 에러 발생시키기)
- **확인 사항**:
  1. 에러 코드 "500" 표시
  2. "서버 오류" 제목 표시
  3. 적절한 에러 메시지 표시

---

### 2.2 폼 필드별 에러 메시지

**테스트 케이스 11: 글 작성 - 제목 필드 에러**
- **조건**: `/write` 페이지 접속
- **절차**:
  1. 제목 필드를 비워두고 내용만 입력
  2. "작성하기" 버튼 클릭
- **확인 사항**:
  1. 제목 입력 필드 아래에 빨간색 에러 메시지 "제목을 입력해주세요." 표시
  2. 제목 필드 테두리가 빨간색(`border-red-300`)으로 변경
  3. 전체 에러 메시지도 상단에 표시

**테스트 케이스 12: 글 작성 - 내용 필드 에러**
- **조건**: `/write` 페이지 접속
- **절차**:
  1. 제목만 입력하고 내용을 비워둠
  2. "작성하기" 버튼 클릭
- **확인 사항**:
  1. 내용 입력 필드 아래에 빨간색 에러 메시지 "내용을 입력해주세요." 표시
  2. 내용 필드 테두리가 빨간색으로 변경

**테스트 케이스 13: 글 작성 - 비밀번호 필드 에러 (익명 사용자)**
- **조건**: `/write` 페이지 접속, 로그아웃 상태
- **절차**:
  1. 제목, 내용 입력
  2. 비밀번호를 3자 이하로 입력 (예: "123")
  3. "작성하기" 버튼 클릭
- **확인 사항**:
  1. 비밀번호 필드 아래에 "비밀번호는 4자 이상으로 입력해주세요." 표시
  2. 비밀번호 확인 필드에 "비밀번호 확인이 일치하지 않습니다." 표시 (비밀번호와 다를 경우)

**테스트 케이스 14: 글 수정 - 필드별 에러**
- **조건**: 익명 게시글 수정 페이지 접속 (로그아웃 상태)
- **절차**:
  1. 제목을 비워두고 "수정하기" 클릭
- **확인 사항**: 제목 필드 아래에 에러 메시지 표시

---

### 2.3 실시간 폼 유효성 검사

**테스트 케이스 15: 제목 실시간 검사**
- **조건**: `/write` 페이지 접속
- **절차**:
  1. 제목 필드에 텍스트 입력
  2. 제목 필드에서 포커스 제거 (blur)
  3. 제목 필드 내용을 모두 삭제
  4. 다시 포커스 제거
- **확인 사항**:
  1. 제목을 비운 후 blur 시 즉시 "제목을 입력해주세요." 에러 메시지 표시
  2. 제목을 다시 입력하면 에러 메시지 사라짐

**테스트 케이스 16: 내용 실시간 검사**
- **조건**: `/write` 페이지 접속
- **절차**:
  1. 내용 필드에 텍스트 입력 후 blur
  2. 내용을 모두 삭제 후 blur
- **확인 사항**: 즉시 "내용을 입력해주세요." 에러 메시지 표시

**테스트 케이스 17: 비밀번호 실시간 검사**
- **조건**: `/write` 페이지 접속, 로그아웃 상태
- **절차**:
  1. 비밀번호 필드에 "123" 입력 후 blur
  2. 비밀번호 확인 필드에 "456" 입력 후 blur
- **확인 사항**:
  1. 비밀번호 필드 blur 시 "비밀번호는 4자 이상으로 입력해주세요." 표시
  2. 비밀번호 확인 필드 blur 시 "비밀번호 확인이 일치하지 않습니다." 표시
  3. 비밀번호를 "1234"로 수정하고 확인도 "1234"로 수정하면 에러 사라짐

## 3. 토스트 알림 시스템 테스트

### 3.1 토스트 컴포넌트 기본 동작

**테스트 케이스 18: 토스트 표시 및 자동 사라짐**
- **조건**: 개발자 도구 콘솔 열기
- **절차**:
  1. 콘솔에서 `import { showToast } from '$lib/stores/toast'; showToast('테스트 메시지', 'success');` 실행
- **확인 사항**:
  1. 우측 상단에 초록색 토스트 메시지 표시
  2. 체크 아이콘 표시
  3. 5초 후 자동으로 사라짐
  4. 닫기 버튼(X) 클릭 시 즉시 사라짐

**테스트 케이스 19: 에러 토스트**
- **조건**: 콘솔 열기
- **절차**: `showToast('에러 메시지', 'error');` 실행
- **확인 사항**: 빨간색 토스트 메시지 표시, X 아이콘 표시

---

### 3.2 글 작성/수정/삭제 토스트

**테스트 케이스 20: 글 작성 성공 토스트**
- **조건**: `/write` 페이지 접속
- **절차**:
  1. 제목, 내용 입력
  2. 로그아웃 상태면 비밀번호 입력
  3. "작성하기" 버튼 클릭
- **확인 사항**:
  1. 성공 시 우측 상단에 "게시글이 작성되었습니다." 초록색 토스트 표시
  2. 게시글 상세 페이지로 리다이렉트

**테스트 케이스 21: 글 작성 실패 토스트**
- **조건**: `/write` 페이지 접속
- **절차**:
  1. 제목만 입력하고 내용 비워둠
  2. "작성하기" 버튼 클릭
- **확인 사항**:
  1. 빨간색 토스트에 "입력한 내용을 확인해주세요." 또는 서버 에러 메시지 표시
  2. 페이지 리다이렉트 없음

**테스트 케이스 22: 글 수정 성공 토스트**
- **조건**: 게시글 수정 페이지 접속
- **절차**:
  1. 제목 또는 내용 수정
  2. "수정하기" 버튼 클릭
- **확인 사항**: "게시글이 수정되었습니다." 초록색 토스트 표시

**테스트 케이스 23: 글 삭제 성공 토스트**
- **조건**: 게시글 상세 페이지 접속, 삭제 권한 있음
- **절차**:
  1. "삭제하기" 버튼 클릭
  2. 확인 다이얼로그에서 "확인" 클릭
  3. 익명 글인 경우 비밀번호 입력
- **확인 사항**:
  1. "게시글이 삭제되었습니다." 초록색 토스트 표시
  2. 게시글 목록 페이지로 리다이렉트

---

### 3.3 댓글 작성 토스트

**테스트 케이스 24: 댓글 작성 성공 토스트**
- **조건**: 게시글 상세 페이지 접속, 로그인 상태
- **절차**:
  1. 댓글 입력 필드에 텍스트 입력
  2. "댓글 작성" 버튼 클릭
- **확인 사항**:
  1. "댓글이 작성되었습니다." 초록색 토스트 표시
  2. 댓글 목록에 새 댓글 즉시 추가 (redirect 없음)

**테스트 케이스 25: 댓글 작성 실패 토스트**
- **조건**: 게시글 상세 페이지 접속, 로그인 상태
- **절차**:
  1. 댓글 입력 필드를 비워둠
  2. "댓글 작성" 버튼 클릭 (또는 서버 에러 발생 시뮬레이션)
- **확인 사항**: 빨간색 토스트에 에러 메시지 표시

---

## 4. 통합 테스트

**테스트 케이스 26: 전체 플로우 테스트**
- **조건**: 로그아웃 상태
- **절차**:
  1. `/write` 페이지 접속
  2. 제목, 내용 입력
  3. 비밀번호 4자 이상 입력, 확인 입력
  4. "작성하기" 클릭
  5. 작성된 게시글에서 "수정하기" 클릭
  6. 내용 수정 후 "수정하기" 클릭
  7. "삭제하기" 클릭, 비밀번호 입력, 확인
- **확인 사항**:
  1. 각 단계마다 적절한 토스트 메시지 표시
  2. 에러 발생 시 필드별 에러 메시지 표시
  3. 모바일에서 버튼 크기 적절한지 확인

**테스트 케이스 27: 로그인 사용자 플로우**
- **조건**: 로그인 상태
- **절차**:
  1. `/write` 페이지 접속
  2. 제목, 내용만 입력 (비밀번호 입력 불필요)
  3. "작성하기" 클릭
  4. 작성된 게시글에서 "수정하기", "삭제하기" 테스트
- **확인 사항**:
  1. 비밀번호 입력 필드가 표시되지 않음
  2. 각 액션마다 토스트 메시지 표시

## 성능 테스트

**테스트 케이스 28: 토스트 다중 표시**
- **조건**: 빠르게 여러 액션 수행
- **절차**: 연속으로 여러 글 작성/수정/삭제
- **확인 사항**: 여러 토스트가 겹치지 않고 순서대로 표시되는지 확인

**테스트 케이스 29: 반응형 전환**
